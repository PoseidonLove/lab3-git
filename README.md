# # Лабораторная работа №3  
## Технологии программирования — Системы контроля версий (VSC)

## Цель работы

Изучение системы контроля версий Git, а также разработка приложения с графическим интерфейсом для анализа статистических данных и выполнения прогнозирования методом скользящей средней.

## Описание проекта

Программа представляет собой графическое приложение для анализа данных о браках и разводах в России за последние 15 лет.

Приложение позволяет:

- загружать данные из Excel файла (.xlsx)
- отображать данные в табличном виде
- строить графики динамики браков и разводов по годам
- анализировать возрастные группы
- выполнять прогнозирование методом скользящей средней
- строить прогноз на последующие N лет
- визуализировать прогноз на отдельном графике
- использовать инструменты масштабирования и перемещения графиков

## Используемые технологии

- Python
- pandas — обработка данных
- matplotlib — построение графиков
- tkinter — графический интерфейс
- openpyxl — чтение Excel файлов

## Описание входных данных

Файл формата `.xlsx` должен содержать следующие столбцы:

- Год
- Браки
- Разводы
- Возраст_мужчин_брак
- Возраст_мужчин_развод
- Возраст_женщин_брак
- Возраст_женщин_развод


## Реализованный функционал

### 1. Загрузка и отображение данных
Пользователь выбирает Excel файл, после чего данные отображаются в таблице.

### 2. Визуализация данных
Построение графика зависимости:
- количества браков по годам
- количества разводов по годам

График поддерживает:
- масштабирование (Zoom)
- перемещение (Pan)
- сброс вида

### 3. Анализ возрастов

Определяются наиболее частые возрастные группы:

- мужчины при вступлении в брак
- мужчины при разводе
- женщины при вступлении в брак
- женщины при разводе

### 4. Прогнозирование

Реализован метод скользящей средней:

\[
MA = \frac{x_1 + x_2 + ... + x_n}{n}
\]

Особенности:
- параметр N задаётся пользователем
- прогноз строится на N будущих лет
- используется накопительное сглаживание

### 5. Визуализация прогноза

На отдельном графике отображаются:
- реальные данные
- прогнозируемые значения
- выделение области прогноза

## Работа с Git

В проекте использовалась система контроля версий Git:

- основная ветка: `main`

Каждое изменение фиксировалось отдельным коммитом с осмысленным описанием.
