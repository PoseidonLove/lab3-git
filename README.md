# Лабораторная работа №3
## Технологии программирования — Системы контроля версий Git

## Цель работы

Изучение принципов работы системы контроля версий Git и получение навыков командной разработки программного обеспечения.  
Разработка графического приложения с графическим интерфейсом для анализа статистических данных и прогнозирования методом скользящей средней.

# Описание проекта

Проект представляет собой общее приложение, объединяющее несколько модулей анализа статистических данных.  
Главное окно программы позволяет запускать отдельные проекты участников команды через единый интерфейс.

В рамках лабораторной работы были реализованы:

- анализ браков и разводов в России;
- анализ статистики преступности в России;
- построение графиков;
- статистический анализ данных;
- прогнозирование методом скользящей средней;
- экспорт графиков;
- работа с Git и GitHub в командном режиме.

# Реализованные варианты

## Вариант №7 — Анализ браков и разводов

Приложение позволяет:

- загружать данные из Excel файла `.xlsx`;
- отображать данные в виде таблицы;
- строить графики браков и разводов по годам;
- определять возрастные группы:
  - мужчин, чаще вступающих в брак;
  - мужчин, чаще разводящихся;
  - женщин, чаще вступающих в брак;
  - женщин, чаще разводящихся;
- выполнять прогнозирование методом скользящей средней;
- строить прогноз на последующие N лет;
- отображать прогноз отдельными линиями;
- использовать масштабирование и перемещение графиков.

## Вариант №13 — Анализ преступности

Приложение позволяет:

- загружать данные из CSV файла;
- отображать статистику преступности в таблице;
- строить графики зависимости уровня преступности от года;
- определять:
  - какой вид преступности снизился больше всего;
  - какой вид преступности снизился меньше всего;
- рассчитывать расширенную статистику:
  - среднее значение;
  - медиану;
  - процент изменения;
- выполнять прогнозирование методом скользящей средней;
- строить прогноз на последующие N лет;
- экспортировать графики в PNG;
- использовать интерактивные инструменты графиков.

# Реализованный функционал

## 1. Загрузка данных

Пользователь выбирает файл через графический интерфейс, после чего данные автоматически загружаются в программу.

## 2. Отображение таблицы

Данные отображаются в удобном табличном виде с помощью `Treeview`.

## 3. Построение графиков

Реализовано построение графиков:

- браков и разводов;
- преступности по видам.

Графики поддерживают:

- Zoom;
- Pan;
- масштабирование;
- перемещение;
- обновление отображения.

## 4. Анализ статистики

### Для варианта №7:

Определяются наиболее частые возрастные категории:

- вступления в брак;
- разводов.

### Для варианта №13:

Определяется:

- максимальное снижение преступности;
- минимальное снижение преступности;
- средние значения;
- медиана;
- процент изменения.

## 5. Прогнозирование

Для обоих вариантов реализован метод скользящей средней.

Формула:

\[
MA = \frac{x_1 + x_2 + ... + x_n}{n}
\]

Особенности реализации:

- пользователь задаёт размер окна N;
- пользователь задаёт количество лет прогноза;
- прогноз строится автоматически;
- используются предыдущие вычисленные значения.

## 6. Визуализация прогноза

Прогноз отображается:

- отдельными пунктирными линиями;
- отдельными цветами;
- с выделением области прогноза.

## 7. Экспорт графиков

В модуле анализа преступности реализован экспорт графиков в формат PNG.

# Принципы SOLID

## Single Responsibility Principle
Каждый класс отвечает только за одну задачу.

## Open/Closed Principle
Приложение можно расширять без изменения существующего кода.

## Dependency Inversion Principle
Интерфейс отделён от логики обработки данных.
